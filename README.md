# ライン職（プロトタイプ・コア最小）

古代遺跡テーマの1人用カードバトル（CPU戦）。ブラウザ完結・サーバー不要で動く。
このリポジトリは「主要メカが動く土台」を作るコア最小スコープ。

## 動かし方

```bash
node server.js
# → http://localhost:3000 をブラウザで開く
```

ESモジュールを使うので `file://` 直開きではなく、上記のローカルサーバー経由で開くこと
（依存ゼロ。`server.js` は標準ライブラリだけで動く）。

エンジンのロジック検証：

```bash
npm test          # = node test/engine.test.mjs（43アサーション）
```

## ゲームの肝

1. **座標干渉する3×3グリッド** — 自分と相手が同じインデックスのマスで向かい合う。
2. **一方通行＋貫通の攻撃** — 殴った側は無傷。敵を倒すと余剰が
   `攻撃力 −（直前の敵HP）＋ 1` ぶんリーダーへ貫通。
3. **ライン完成** — 縦横斜め3マスを自分の石像で揃えると、そのライン上の石像が攻撃力+1
   （交点は重複で複数+1。盤面依存で常に動的計算）。

詳細仕様は要件定義のとおり。後攻には「起動の石」(マナ+1)、先攻はターン1ドローなし。

## 構成

```
server.js              ゼロ依存の静的サーバー（Railwayエントリ兼用）
public/
  index.html, style.css
  js/
    cards.js           カード定義（単一の真実のソース）
    engine.js          ★純粋ゲームロジック（DOM/乱数源/描画に非依存）
    ai.js              CPU（greedy）
    ui.js              描画と入力（engineを呼ぶだけ）
    main.js            起動
test/engine.test.mjs   engineのNodeテスト
```

### 設計方針（将来のオンライン化のため）

`engine.js` は **DOMにも描画にも一切依存しない純粋モジュール**。状態はプレーンオブジェクトで、
`applyAction(state, action)` だけが状態を進める。`ui.js` は state を読んで描き、入力を action に変換するだけ。

→ Railway でのオンラインマルチプレイに拡張するときは：
- `server.js` に WebSocket を足し、サーバー側で `engine` を **サーバー権威** で動かす
- クライアントは action を送信し、サーバーから配信される state を描画する
- `ui.js` の `applyAction(...)` 呼び出しを「サーバーへ action 送信」に差し替えるだけで移行できる

乱数もシード可能（`mulberry32`）にしてあるので、サーバー権威でも決定的に再現できる。

## スコープ外（次フェーズ）

凝ったカード効果（ドロー・トークン生成・召喚時移動・バーン等）、遺跡ビジュアルの作り込み、
デッキ構築、オンライン対戦。数値（リーダーHP20・貫通+1・スタッツ）は実プレイ後に調整。
