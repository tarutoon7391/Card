# ライン職（プロトタイプ・コア最小）

古代遺跡テーマのカードバトル。**CPU戦（オフライン）** と **オンライン2人対戦** に対応。
ブラウザ完結・**依存ゼロ**（`server.js` は Node 標準ライブラリだけで動く）。

**🚀 本番（Railway）: https://card-production-0042.up.railway.app**

## 動かし方

```bash
node server.js
# → http://localhost:3000 をブラウザで開く
```

ESモジュールを使うので `file://` 直開きではなく、上記のローカルサーバー経由で開くこと。
ロビーから次の3つを選べる：

- **CPU と対戦** … オフライン。サーバー不要（GitHub Pages でも動く）。
- **部屋を作る / コードで参加** … 4文字の合言葉（または招待URL `?room=XXXX`）で友達と対戦。
- **クイックマッチ** … 待機列で他の待機者と自動マッチング。

> オンライン対戦は WebSocket サーバーが要るので **GitHub Pages（静的）では動かない**。
> `node server.js` か Railway 等のサーバー上で動かすこと（`server.js` がそのエントリ）。

エンジン／直列化のロジック検証：

```bash
npm test          # = node test/engine.test.mjs（56アサーション）
```

## ゲームの肝

1. **座標干渉する3×3グリッド** — 自分と相手が同じインデックスのマスで向かい合う。
2. **一方通行＋貫通の攻撃** — 殴った側は無傷。敵を倒すと余剰が
   `攻撃力 −（直前の敵HP）＋ 1` ぶんリーダーへ貫通。
3. **ライン完成** — 縦横斜め3マスを自分の石像で揃えると、そのライン上の石像が攻撃力+1
   （交点は重複で複数+1。盤面依存で常に動的計算）。

詳細仕様は要件定義のとおり。後攻には「起動の石」(マナ+1)、先攻はターン1ドローなし。

## 構成

```
server.js              静的配信 ＋ オンライン対戦サーバー（ゼロ依存・Railwayエントリ兼用）
public/
  index.html, style.css
  js/
    cards.js           カード定義（単一の真実のソース）
    engine.js          ★純粋ゲームロジック（DOM/乱数源/描画に非依存）＋ serializeFor
    ai.js              CPU（greedy）
    ui.js              盤面の描画と入力（cpu/online 両モード共通）
    net.js             サーバーとの WebSocket 接続ラッパ
    app.js             ロビー／接続オーケストレーション（モード選択・招待URL）
    main.js            起動
test/engine.test.mjs   engine と serializeFor の Node テスト
```

### オンライン化の設計（実装済み）

`engine.js` は **DOMにも描画にも一切依存しない純粋モジュール**。状態はプレーンオブジェクトで、
`applyAction(state, action)` だけが状態を進める。この純粋さを活かして、オンラインは
**サーバー権威**で実装してある：

- `server.js` が Node 標準ライブラリだけで **WebSocket（RFC 6455）を自前実装**し、
  サーバー側で `engine` を権威として動かす（外部ライブラリ依存ゼロを維持）。
- クライアント（`ui.js`）は手番に action を送るだけ。サーバーは `applyAction` で状態を進め、
  各プレイヤーへ配信する。`ui.js` の適用先がモードで切り替わる
  （`cpu`＝ローカル `applyAction` / `online`＝サーバーへ送信）。
- **隠匿情報**は `engine.js` の `serializeFor(state, seat)` が席ごとに伏せて配信する
  （相手の手札・両者の山札の中身は見えない。`seed` も送らない）。

乱数はシード可能（`mulberry32`）。`seed` はサーバーだけが保持するので、引きを予測されない。

### マッチメイク

- **ルームコード**：作成者が席0、参加者が席1。4文字コード or 招待URL（`?room=XXXX`）で合流。
- **クイックマッチ**：待機者が1人いれば即マッチ、いなければ待機列に入る。
- 先攻／後攻は対局ごとにランダム（`createGame` の乱数）。切断は相手へ通知してその対局を終了。
